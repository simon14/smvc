Detailed Description

A blog controller to display a blog-like list of all content labelled as "post".

A user controller to manage content.

To manage and analyse all modules of Lydia.

A page controller to display a page, for example an about-page, displays content labelled as "page".

A test controller for themes.

A utility class to easy creating and handling of forms

A form to manage content.

A form for creating a new user.

A form to login the user profile.

A wrapper for HTMLPurifier by Edward Z. Yang, http://htmlpurifier.org/

A model for content stored in database.

A model for managing Lydia modules.

Interface for class that can be installed / updated / uninstalled.

The method should implement the needed actions for installing, updating or deinstalling the module. The module can be the class that implements the interface and its related classes, functions, etc. Ordinary actions to carry out is creating tables in database, checking writable directory for cache-files, checking pre-conditions that PHP-extensions are available and the right versions of software are installed.
